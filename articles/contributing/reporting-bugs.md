# <a name="reporting-bugs"></a>Raportowanie usterek #

Jeśli uważasz, że znaleźliśmy usterkę w składniku zestawu Quantum Development Kit, bardzo doceniamy raport.
Następnie ktoś inny może być zoptymalizowaniem z tym samym problemem, jak również: informowanie nas pomaga nam w rozwiązaniu tego problemu dla wszystkich użytkowników.
Pierwszym krokiem w raportowaniu usterki jest rozpoczęcie od przejrzenia istniejących problemów występujących w każdym repozytorium, ponieważ jest to bardzo możliwe, że ktoś inny zgłosił ten sam problem.

Jeśli problem został już znaleziony wcześniej, może być przydatne podanie dodatkowych informacji lub kontekstu, które mogą pomóc nam w zdiagnozowaniu i rozwiązaniu problemu.
W takim przypadku możesz bezpłatnie uczestniczyć w dyskusjach dotyczących problemu.

Jeśli usterka nie została wcześniej zgłoszona, dowiesz się, w jaki sposób otworzysz nowy problem w repozytorium dla danego składnika.
Podczas tworzenia nowego problemu zostanie wyświetlony monit z szablonem sugerującym pewne informacje, które często pomagają w tworzeniu raportów o usterkach:

- Kroki prowadzące do odtworzenia problemu
- Oczekiwane miejsce
- Rzeczywiste zaszło
- Informacje o środowisku oprogramowania (np.: OS, .NET Core i Quantum Development Kit)

Możesz również [utworzyć żądanie ściągnięcia](https://help.github.com/articles/about-pull-requests/) , aby naprawić usterkę bezpośrednio, jeśli jest bardzo prosta i nie jest to dyskusja (na przykład literówka).
